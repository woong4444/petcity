# 상각_07-19_20 SQL 적용 순서 및 기능 대응표

## 원칙

- 8081 팀장 원본의 0~19번 SQL은 변경하지 않습니다.
- 8080에서 추가한 기능의 DB 변경만 20번 이후 파일로 분리했습니다.
- 아래 순서대로 필요한 파일만 실행합니다.

## 실행 대상

### 20번 — 필수

`20_상각_07-19_20 sql 수정_APP_MEMBER 탈퇴 복구코드 컬럼.sql`

- 기존 `APP_MEMBER`에 `RECOVERY_TOKEN_HASH`를 추가합니다.
- 구버전 DB에 `DELETE_REASON`이 없을 때만 함께 추가합니다.
- 복구코드 원문이 아닌 암호화 해시를 저장합니다.
- 같은 스크립트를 다시 실행해도 이미 존재하는 컬럼은 추가하지 않습니다.

### 21번 — 병원 이미지 데이터 적용 시 실행

`21_상각_07-19_20 sql 수정_병원 이미지 경로 매칭.sql`

- 팀장이 제공한 병원 이미지 파일과 `HOSPITAL.IMAGE_URL`을 병원 번호 기준으로 연결합니다.
- 회원 탈퇴·복구 기능과는 독립된 데이터 보정 SQL입니다.
- 운영 데이터에 적용하기 전에 파일 경로와 병원 ID를 확인합니다.

### 22번 — 보안이력 기능 사용 시 필수

`22_상각_07-19_20 sql 추가_회원 탈퇴 복구 보안이력 1년보관.sql`

- `MEMBER_SECURITY_AUDIT` 테이블을 새로 생성합니다.
- `SEQ_MEMBER_SECURITY_AUDIT` 시퀀스와 조회용 인덱스를 생성합니다.
- 탈퇴 요청, 복구 시도, 복구 성공·실패, 복구코드 재발급 이력을 1년간 보관합니다.
- 복구코드 및 이메일 인증번호 원문은 저장하지 않습니다.

## 별도 SQL이 필요하지 않은 최근 기능

다음 기능은 20~22번 사용하며 추가 DB 변경이 없습니다.
SQL은 더 실행하지 말고, 코드만 최신 버전으로 받으세요

- 회원가입 화면에서 기존 SNS 회원의 중복가입 안내
- 기존 SNS 회원 안내 후 10초 뒤 메인 화면 이동
- SNS 탈퇴 시 비밀번호 입력 생략
- SNS 탈퇴 대기 계정 로그인 차단 및 사유 안내
- 가입 시 확인된 등록 이메일로 인증번호 단계 없이 새 복구코드 직접 발송
- 일반·SNS 회원의 등록 이메일과 복구코드 일치 확인 후 즉시 복구
- 관리자 회원 상세의 탈퇴 사유 표시
- 아이디 찾기, 비밀번호 재설정, 마이페이지

## 적용 확인 SQL

```sql
SELECT COLUMN_NAME, DATA_TYPE, DATA_LENGTH
FROM USER_TAB_COLUMNS
WHERE TABLE_NAME = 'APP_MEMBER'
  AND COLUMN_NAME IN ('RECOVERY_TOKEN_HASH', 'DELETE_REASON');

SELECT TABLE_NAME
FROM USER_TABLES
WHERE TABLE_NAME = 'MEMBER_SECURITY_AUDIT';

SELECT SEQUENCE_NAME
FROM USER_SEQUENCES
WHERE SEQUENCE_NAME = 'SEQ_MEMBER_SECURITY_AUDIT';
```
